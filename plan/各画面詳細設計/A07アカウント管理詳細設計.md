# A07 アカウント管理詳細設計

## 1. 画面概要
管理システムのログインアカウント自体の管理を行う（Admin 権限必須）。

## 2. 必要なフォームデータ
- `login_name`: ログインID
- `email`: メールアドレス
- `password`: パスワード（新規作成または変更時）
- `role`: `admin` / `creator`
- `status`: `active` / `banned`

## 3. I/O（API フォーマット）

### 3.1. アカウント一覧取得
- **メソッド**: `GET`
- **エンドポイント**: `/api/admin/accounts`

#### Response JSON
```json
{
  "accounts": [
    {
      "account_id": 1,
      "login_name": "admin_user",
      "email": "admin@example.com",
      "role": "admin",
      "status": "active",
      "last_login_at": "2026-01-18T10:00:00Z"
    },
    {
      "account_id": 2,
      "login_name": "creator_01",
      "email": "c01@example.com",
      "role": "creator",
      "status": "active",
      "last_login_at": "2026-01-17T15:00:00Z"
    }
  ]
}
```

### 3.2. アカウント作成
- **メソッド**: `POST`
- **エンドポイント**: `/api/admin/accounts`

#### Request JSON
```json
{
  "login_name": "new_user",
  "email": "new@example.com",
  "password": "secure_password",
  "role": "creator"
}
```

### 3.3. アカウント情報編集（ロール変更・BAN等）
- **メソッド**: `POST`
- **エンドポイント**: `/api/admin/accounts/{id}`

#### Request JSON
```json
{
  "email": "updated@example.com",
  "role": "admin",
  "status": "active"
}
```

### 3.4. パスワードリセット
- **メソッド**: `PATCH`
- **エンドポイント**: `/api/admin/accounts/{id}/password`

#### Request JSON
```json
{
  "new_password": "new_secure_password"
}
```
