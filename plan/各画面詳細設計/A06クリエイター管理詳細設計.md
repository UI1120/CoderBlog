# A06 クリエイター管理詳細設計

## 1. 画面概要

執筆者（個人クリエイター）およびグループクリエイター情報の登録、プロフィール編集、およびメンバーシップ管理を行う。
※個人クリエイターの登録には、あらかじめ A07 で発行された `account_id` が必要となる。通常、アカウント発行時に自動的に個人クリエイターレコードも生成される運用を想定する。

## 2. 必要なフォームデータ

- `display_name`: 表示名
- `creator_type`: `individual` (個人) / `group` (グループ)
- `profile`: 自己紹介（Markdown または プレーンテキスト）
- `member_creator_ids`: 所属メンバーの ID リスト（グループクリエイターのみ）

## 3. I/O（API フォーマット）

### 3.1. クリエイター一覧取得

- **メソッド**: `GET`
- **エンドポイント**: `/api/admin/creators`

#### Response JSON

```json
{
  "creators": [
    {
      "creator_id": 1,
      "display_name": "山田 太郎",
      "creator_type": "individual",
      "profile": "ロボットエンジニアです。"
    },
    {
      "creator_id": 10,
      "display_name": "ロボット工作部",
      "creator_type": "group",
      "profile": "有志でロボットを作っています。",
      "members": [
        { "creator_id": 1, "display_name": "山田 太郎" },
        { "creator_id": 2, "display_name": "田中 花子" }
      ]
    }
  ]
}
```

### 3.2. クリエイター情報更新

- **メソッド**: `POST`
- **エンドポイント**: `/api/admin/creators/{id}`

#### Request JSON

```json
{
  "display_name": "更新後の名称",
  "profile": "プロフィールの更新内容...",
  "member_creator_ids": [1, 2, 5]
}
```

### 3.3. クリエイター新規登録

- **メソッド**: `POST`
- **エンドポイント**: `/api/admin/creators`

#### Request JSON

```json
{
  "display_name": "新規クリエイター",
  "creator_type": "individual",
  "profile": "こんにちは...",
  "account_id": 5
}
```
