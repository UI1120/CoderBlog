# P04 プロジェクト詳細設計

## 1. 画面概要

プロジェクトの一覧ページ（List）および個別のプロジェクト詳細ページ（Detail）を提供する。

- **List Mode**: `/project` でアクセス。全プロジェクトをカード表示。カテゴリフィルタあり。
- **Detail Mode**: `/project?pid={id}` でアクセス。特定のプロジェクトの説明と紐づく記事一覧を表示。

## 2. 必要な表示データ

### List Mode

- **プロジェクト一覧**: 全プロジェクトのメタデータ。

### Detail Mode

- **プロジェクト詳細**: 名前、説明、サムネイル。
- **関連記事**: そのプロジェクトに紐づく記事リスト。

## 3. I/O (API フォーマット)

### 3.1. プロジェクト一覧取得

- **メソッド**: `GET`
- **エンドポイント**: `/api/projects`
- **Response Example**:

```json
[
  {
    "project_id": 1,
    "project_name": "Game Dev",
    "description": "Making games...",
    "category_id": 1,
    "category": "Entertainment",
    "thumbnail": "https://..."
  }
]
```

### 3.2. プロジェクト詳細取得

- **メソッド**: `GET`
- **エンドポイント**: `/api/projects`
- **Query Parameters**: `?pid={id}`
- **Response Example**:

```json
{
  "project_name": "Game Dev",
  "description": "Full description...",
  "thumbnail": "https://..."
}
```

※実装上は一覧と同じエンドポイントを使用し、PID指定で単一オブジェクト（またはフィルタ済みリストの先頭）を返却する形式となっている。

### 3.3. 関連記事取得

- **メソッド**: `GET`
- **エンドポイント**: `/api/articles`
- **Query Parameters**:
  - `pid`: Project ID
  - `page`: ページ番号 (Default: 1)
  - `limit`: 1ページあたりの件数 (Default: 12)
- **Response Example**:

```json
{
  "articles": [
    {
      "article_id": 101,
      "title": "Project Update #1",
      "summary": "...",
      "thumbnail": "...",
      "published_date": "2026-02-01"
    }
  ],
  "total": 5,
  "page": 1,
  "limit": 12
}
```
