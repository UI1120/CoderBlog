# A05 分類・マスタ管理詳細設計

## 1. 画面概要
システム内で使用されるカテゴリ、タグ、プロジェクト情報の管理（作成・編集・削除）を行う。

## 2. 必要なフォームデータ

### 2.1. カテゴリ (Categories)
- `category_name`: カテゴリ名

### 2.2. タグ (Tags)
- `tag_name`: タグ名

### 2.3. プロジェクト (Projects)
- `project_name`: プロジェクト名
- `description`: プロジェクト説明文

## 3. I/O（API フォーマット）

### 3.1. マスタ一覧取得 (例: カテゴリ)
- **メソッド**: `GET`
- **エンドポイント**: `/api/admin/categories`

#### Response JSON
```json
{
  "categories": [
    { "category_id": 1, "category_name": "ロボット" },
    { "category_id": 2, "category_name": "Web開発" }
  ]
}
```

### 3.2. マスタ登録 (例: タグ)
- **メソッド**: `POST`
- **エンドポイント**: `/api/admin/tags`

#### Request JSON
```json
{
  "tag_name": "NewTag"
}
```

### 3.3. マスタ更新 (例: プロジェクト)
- **メソッド**: `POST`
- **エンドポイント**: `/api/admin/projects/{id}`

#### Request JSON
```json
{
  "project_name": "Updated project",
  "description": "Updated description..."
}
```

### 3.4. マスタ削除
- **メソッド**: `POST`
- **エンドポイント**: `/api/admin/{resource}/{id}`
- ※ `resource` は `categories`, `tags`, `projects` のいずれか。
