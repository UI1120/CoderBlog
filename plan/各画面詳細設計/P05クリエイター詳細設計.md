# P05 クリエイター詳細設計

## 1. 画面概要

クリエイター（個人・グループ）の一覧ページ（List）および詳細プロフィールページ（Detail）を提供する。

- **List Mode**: `/creator` でアクセス。`individual` / `group` タブで切り替え表示。
- **Detail Mode**: `/creator?cid={id}` (個人) または `?gid={id}` (グループ) でアクセス。

## 2. 必要な表示データ

### List Mode

- **クリエイター一覧**: 表示名、プロフィール、タイプ、アイコン。

### Detail Mode

- **クリエイター詳細**: 詳細プロフィール、所属メンバー（グループの場合）。
- **執筆記事**: そのクリエイターが執筆した記事リスト。

## 3. I/O (API フォーマット)

### 3.1. クリエイター一覧取得

- **メソッド**: `GET`
- **エンドポイント**: `/api/creators`
- **Response Example**:

```json
[
  {
    "creator_id": 1,
    "display_name": "Yamada",
    "creator_type": "individual",
    "profile": "Software Eng.",
    "icon_path": "https://..."
  },
  {
    "creator_id": 10,
    "name": "RoboGroup",
    "creator_type": "group",
    "members": [...]
  }
]
```

### 3.2. クリエイター詳細取得

- **メソッド**: `GET`
- **エンドポイント**: `/api/creators`
- **Query Parameters**: `?cid={id}` or `?gid={id}`
- **Response Example**:

```json
{
  "creator_id": 1,
  "display_name": "Yamada",
  "profile": "...",
  "icon_path": "..."
}
```

### 3.3. 執筆記事取得

- **メソッド**: `GET`
- **エンドポイント**: `/api/articles`
- **Query Parameters**: `?cid={id}` (writer_id) or `?gid={id}` (group_id)
- **Response Example**:

```json
[
  { "article_id": 1, "title": "My First Post", ... }
]
```
