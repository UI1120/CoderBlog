# A01 ログインシステム詳細設計

## 1. 画面概要
管理画面（ダッシュボードやエディター）にアクセスするための認証画面。

## 2. 必要なフォームデータ
- `login_name`: ユーザー名（または ID）
- `password`: パスワード

## 3. I/O（API フォーマット）

### 3.1. ログイン実行
- **メソッド**: `POST`
- **エンドポイント**: `/api/auth/login`

#### Request JSON
```json
{
  "login_name": "user123",
  "password": "password_here"
}
```

#### Response JSON (Success)
```json
{
  "status": "success",
  "message": "Logged in successfully",
  "account": {
    "account_id": 1,
    "login_name": "user123",
    "display_name": "山田 太郎",
    "role": "admin"
  }
}
```

#### Response JSON (Error)
```json
{
  "status": "error",
  "message": "Invalid username or password"
}
```

### 3.2. ログイン状態確認（ページロード時に使用）
- **メソッド**: `GET`
- **エンドポイント**: `/api/auth/me`

#### Response JSON (Success)
```json
{
  "is_authenticated": true,
  "account": {
    "account_id": 1,
    "login_name": "user123",
    "display_name": "山田 太郎",
    "role": "admin"
  }
}
```

#### Response JSON (Unauthenticated)
```json
{
  "is_authenticated": false
}
```
